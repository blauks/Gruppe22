Administrative details

The European Partnership for Responsible Minerals (EPRM) is a multi stakeholder partnership that is aiming to improve the working and social conditions of those involved in the extraction and processing of 3TG minerals. The proposed case is to redesign their current website to better communicate relevant information to viewers. Whilst the case is based on a real scenario, EPRM itself is not involved, and the produced deliverables will not be employed by the partnership. Instead, the case has been put forward by a group member, Bram Goosink, who is an ex-employee of the EPRM.

Our website is located at http://folk.ntnu.no/jonatbr

Changes

Changes to files/file structure

  • Added more specific CSS and JS files for each page/functionality, instead of just using one big one. This increases maintainability and readability.
  • Moved video file into same folder as the images

Changes in about.html

  • Added transition effects to the buttons and slight changes to the margins to make it more suitable format for mobile devices.

Changes in application.html

  • Added a transition for the background of the bars when you hover over them, to indicate that they are clickable. Also made the cursor change to pointer when you hover over the bars.

Changes in CallsForProposals.html

  • Visual design change on buttons, no longer a frame around the buttons, and the text fades in on hover

Changes in News.html

  • No image changing according to selected article due to lack of such images from our client. Using a single default image instead
  • No link to each individual newsletter on the web page as these will be sent as a pdf upon subscribing.

Changes to minimal requirements/ Javascript

  • Handling of buttons and button animations in CallsForProposals.html was done using Javascript
  • Used CSS to gray out and colour the Get Involved buttons when you hover over them instead of JavaScript.
  • the structure section in the about page in about.html got a fade effect instead of a slide effect. Furthermore the radio button got realised with radio button like box shadow.

How we reached modularity requirements

We used different JavaScript files to reach a modular website. The header.js and footer.js are used on all the different sites to include the navbar on the top of the page and the footer on the bottom of the page.

Minimum Requirements

Kristian fulfilled his minimum requirements by making the button handlers located in js/handlers.js, this minimum requirement was added after P2, see changes Section

Jonathan did all of his minimum requirements, but instead of using javascript to change the colour of the get involved buttons when you hover over them he used CSS.

Line fulfilled all minimum requirements of a calendar displaying the date at which an article was posted, an animated subscription bar, an interchanging map on the projects page and images changing based on the selected article.

Nicolai also fulfilled his minimum requirements, by using javascript to both generate and interact(such as hover effects and links) with the icons in the EPRM members list. Furthermore, Javascript was also used in the structure view to do the text transition and the radio-button.
